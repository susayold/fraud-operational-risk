# Recruiter Talking Points

## 30-second story

I built a fraud and operational-risk framework that connects an observed 284.8K-row fraud benchmark to threshold economics, then uses a separately labeled synthetic environment for rules, capacity, incidents, KRIs and action ownership. The champion reaches 0.7786 PR-AUC, but fraud-amount recall remains an Amber exception, so the recommendation includes complementary high-value controls rather than declaring victory from one model metric.

## Senior-risk answer

Fraud strategy is a constrained operating decision. A higher score is valuable only if the selected threshold, alert capacity, customer friction, amount capture and fallback controls fit risk appetite and operations.

## Honest limitation answer

With production data I would add event-time leakage controls, device/merchant/customer features, confirmed loss and recovery, investigator outcomes, concept drift, control shadow testing and independent validation.
