# Fraud Detection, Controls & Operational Risk

## [OPEN LIVE PROJECT](https://susayold.github.io/fraud-operational-risk/)

Interactive website with English and Vietnamese views. The live project is the recommended starting point.

This project combines an observed imbalanced fraud benchmark with a separately labelled synthetic controls environment, then connects detection performance to threshold economics, investigation capacity, incidents and operational-risk governance.

## Key Results

- 284,807 observed transactions and 492 observed fraud cases assessed.
- 0.7786 champion test PR-AUC.
- 84.93% test fraud recall.
- 69.36% test fraud-amount recall.
- 228/228 analytical, testing and governance checks passed.

## What The Project Demonstrates

- Fraud modeling under extreme class imbalance.
- PR-AUC, precision, recall, amount capture and calibration assessment.
- Capacity-aware threshold economics and net-benefit analysis.
- Interpretable transaction rules, reason codes and hybrid routing.
- Alert queues, SLA/backlog stress, incidents, RCA and residual-risk actions.
- Clear separation of observed evidence, synthetic controls and dispute labels.

## Analytical Boundary

Observed PCA-transformed transactions support benchmark model performance. Synthetic transaction features support interpretable rule, routing and operating-control tests only. The project is not presented as a live payment control, AML system or production fraud platform.
